  <what-built>
Complete chat widget built and served from the local dev server. The widget is a minified IIFE loaded via a single script tag. It uses Shadow DOM for isolation, renders a floating chat bubble (bottom-right), expands into a full chat panel, consumes SSE streams from the Phase 3 AI engine, and persists sessions in localStorage.
  </what-built>
  <how-to-verify>
Open http://localhost:4267/widget-test.html in your browser and work through this checklist:

**Visual / Layout:**
1. Floating dark circle (56px) appears in the bottom-right corner of the page
2. Page content loads normally — widget does NOT block or shift page layout
3. On first load (clear localStorage first via DevTools > Application > Local Storage > Clear All): bubble pulses twice after ~1.2 seconds

**Panel open/close:**
4. Click the bubble: chat panel opens (animated, bottom-right, above bubble)
5. Panel header shows "AI Assistant" with a dark circle avatar labeled "A"
6. Empty state shows "Hi! How can I help?" centered in the message area
7. Input field is focused automatically when panel opens
8. Bubble icon has changed to an X (close icon)
9. Click the X bubble again: panel closes, bubble icon returns to chat icon

**Sending a message:**
10. Open the panel, type a message (e.g., "Hello"), press Enter
11. Your message appears as a dark bubble on the right
12. Three animated dots appear on the left (typing indicator)
13. After a few seconds, the full AI response appears as a light bubble on the left
14. Typing dots are gone, input field is cleared and re-enabled

**Session persistence:**
15. Send a message, then reload the page (F5)
16. Open the panel again, send another message
17. Confirm: the same sessionId is reused (check Network tab in DevTools — the POST body includes `sessionId`)

**Async / GTM compatibility:**
18. In the page source, confirm the script tag has `async` attribute
19. Confirm the page title and body text appear BEFORE the widget bubble (no render blocking)

**Error state (optional):**
20. Temporarily set `data-token="invalid-token"` in the HTML, reload, send a message
21. Confirm: error message "This chat is unavailable right now." appears in the chat

Type "approved" if all checks pass, or describe any issues you observe.
