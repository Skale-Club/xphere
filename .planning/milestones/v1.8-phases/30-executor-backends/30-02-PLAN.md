# Plan 30-02: Custom Webhook Executor

## Objective

Implement the `custom_webhook` executor so tool_configs with that action_type fire a configurable HTTP request. The URL, method, headers, and body template are read from `tool_config.config` JSONB. `{{param_name}}` placeholders in the body template are replaced with matching tool call parameter values before the request is sent. A 10-second timeout prevents hanging. The executor is wired into `execute-action.ts` replacing the current stub.

## Context

- `src/lib/action-engine/execute-action.ts` — dispatcher that has `case 'custom_webhook': throw new Error('Unsupported action type: custom_webhook')`
- `src/lib/google-contacts/create-contact.ts` — executor pattern to replicate (single-line return, no newlines in result string)
- `src/types/database.ts` — `tool_configs.config` is `Json` (arbitrary JSONB); `action_type` enum already includes `'custom_webhook'`
- `src/app/api/vapi/tools/route.ts` — call site for executeAction
- `src/lib/manychat/dispatch-event.ts` — second call site for executeAction
- No new DB migrations required — action_type and config column already exist

## Tasks

### Task 1: Create custom-webhook executor module
**type:** auto
**files:** `src/lib/custom-webhook/execute-webhook.ts`

Create the `src/lib/custom-webhook/` directory and implement `executeWebhook(params, config)`.

Config shape (read from `tool_config.config` JSONB):
```ts
interface WebhookConfig {
  url: string
  method?: 'GET' | 'POST' | 'PUT' | 'PATCH' | 'DELETE'  // default 'POST'
  headers?: Record<string, string>
  body?: string  // template with {{param_name}} placeholders
}
```

Implementation requirements:
- Parse and validate `url` from config; throw `'custom_webhook: missing url in tool config'` if absent
- Default method to `'POST'` if not specified
- Replace `{{param_name}}` in body template with `String(params[param_name] ?? '')` for each key in params
- Set `AbortController` with 10-second timeout
- On success: return single-line string `'Webhook OK status=${status} body=${truncated}'` (body truncated to 500 chars, no newlines — Vapi parser constraint)
- On non-2xx: return `'Webhook error status=${status} body=${truncated}'` (not throw — executor reports error as result string, action engine catches as needed)
- Propagate `AbortError` (timeout) directly so the caller can detect it via `err.name === 'AbortError'`
- No newlines in any returned string (replace with space)

### Task 2: Wire custom_webhook into execute-action.ts
**type:** auto
**files:** `src/lib/action-engine/execute-action.ts`

- Import `executeWebhook` from `@/lib/custom-webhook/execute-webhook`
- Replace the `case 'custom_webhook'` stub with a real dispatch:
  ```ts
  case 'custom_webhook': {
    const webhookConfig = (params.__toolConfig as Record<string, unknown> | undefined) ?? {}
    // config comes from ctx or toolConfig — resolve from ctx pattern
    if (!ctx?.supabase || !ctx?.organizationId) {
      throw new Error('custom_webhook requires ctx')
    }
    return executeWebhook(params, toolConfig)
  }
  ```
  
  Wait — the execute-action signature doesn't have toolConfig directly available. The config must be passed differently.

  **Correct approach:** `executeAction` receives `params` and `credentials` but NOT the raw tool_config.config. The config JSONB must be passed in. 

  Add a `toolConfig?: import('@/types/database').Json` field to `ActionContext`, then pass it from the call sites. Update `execute-action.ts` to accept `ctx.toolConfig` and forward it to `executeWebhook`.

- In `case 'custom_webhook'`:
  ```ts
  case 'custom_webhook': {
    if (!ctx?.toolConfig) {
      throw new Error('custom_webhook requires ctx.toolConfig (the tool_config.config JSONB)')
    }
    return executeWebhook(params, ctx.toolConfig)
  }
  ```

### Task 3: Update call sites to pass toolConfig in ctx
**type:** auto
**files:**
- `src/app/api/vapi/tools/route.ts`
- `src/lib/manychat/dispatch-event.ts`

Both call sites already have `toolConfig.config` available. Pass it as `ctx.toolConfig`:

In `route.ts`:
```ts
result = await executeAction(toolConfig.action_type, args, credentials, {
  organizationId: orgId,
  supabase,
  toolConfig: toolConfig.config,
})
```

In `dispatch-event.ts`:
```ts
result = await executeAction(
  tool.action_type,
  input.payload,
  credentials,
  { organizationId: input.orgId, supabase, toolConfig: tool.config }
)
```
