## Overview

Promotes **agent** to a first-class entity in Operator for chat channels (web widget, WhatsApp, Messenger, Instagram, ManyChat, Telegram). Adds multi-agent delegation with loop detection, idempotency, intersection authorization, and a full observability dashboard. Voice (Vapi) is explicitly untouched — `assistant_mappings`, `/api/vapi/*`, and `resolveTool(orgId, toolName)` unchanged.

## Key Accomplishments

1. **Agent entity shipped** — `agents` table with per-org prompt, model, tool scoping, channel overrides JSONB; every existing org backfilled with a "Main Agent" byte-identical to v1.4 chat behavior
2. **Channel-agnostic runtime** — `runAgent()` entry point replaces monolithic `chat/stream.ts`; web widget, ManyChat, Meta all consume the same runtime; 5 channel adapters normalize output per wire format
3. **Multi-agent delegation** — partner tool injection, recursive `runAgent()` with 3-tier handoff payload, `MAX_DELEGATION_DEPTH=2`, visited-set loop detection, intersection authz across the full chain
4. **Idempotency** — SHA-256 key wrappers on `create_appointment`, `send_sms`, `create_contact`, `custom_webhook`; same key fired twice → one execution, byte-identical response
5. **Prompt versioning UX** — DB trigger snapshots every `agents.system_prompt` change; runtime reads `active_prompt_version_id` exclusively; draft/publish flow + inline LCS diff viewer + one-click rollback
6. **Observability UI** — per-agent metrics widgets (p50/p95/cost/tool success), per-org cost ticker with cap alert, delegation tree drill-in, invocations list with filters, agent badge on assistant messages
7. **Agent CRUD dashboard** — `/dashboard/agents` list/create/edit with tool picker (v1.5 folder hierarchy), channel defaults editor, channel overrides form, partner agents panel
8. **Multi-channel playground** — `/dashboard/agents/[id]/playground` with 6-channel selector, streaming inline, tool-call display, delegation badges; `mode='playground'` excludes from production metrics
9. **Google SSO + Team Invites** — `org_invites` allowlist, OAuth callback, `/dashboard/members` page; 42-02 (Google Cloud Console config) deferred to operator setup

## Key Decisions

| Decision | Outcome |
|----------|---------|
| **agent** naming (not "bot" or "assistant") | Avoids collision with Vapi Assistants and OpenAI Assistants |
| Voice (Vapi) explicitly untouched | `resolveTool(orgId, toolName)` unchanged; chat catches up, not unifies |
| `ai@^6` SDK adopted | Spike confirmed lower latency and cleaner streaming vs custom fetch |
| Intersection authz (all chain members must have tool) | Prevents privilege escalation in multi-agent flows |
| Handoff payload: 3-tier structure, no raw history | Prevents prompt injection via history forwarding |
| `active_prompt_version_id` → never fall back to `agents.system_prompt` | Null pointer on missing version (forces proper seeding) |
| Playground tagged `mode='playground'` | Excluded from production cost caps and observability widgets |
| Google SSO allowlist per org (not domain restriction) | Operator is multi-tenant; emails from different orgs coexist |
| `org_members` table reused (not new `memberships`) | Already exists from migration 001 |
