From src/app/api/stripe/webhook/route.ts:
```typescript
export const runtime = 'nodejs'
export async function POST(request: Request): Promise<Response>

// Response contract:
//   invalid / missing signature      -> 400
//   duplicate event already handled  -> 200 { received: true, duplicate: true }
//   handled successfully             -> 200 { received: true }
//   transient processing failure     -> 500

const SUPPORTED_EVENTS = new Set([
  'checkout.session.completed',
  'customer.subscription.created',
  'customer.subscription.updated',
  'customer.subscription.deleted',
  'invoice.paid',
  'invoice.payment_failed',
])
```

Exact sequence of calls the route makes (this is what must be mocked):
1. `getStripe().webhooks.constructEvent(rawBody, signature, secret)` — DO NOT mock this away; use a real `new Stripe(...)` instance in the test to sign, so this real code path is genuinely exercised.
2. `supabase.from('billing_events').upsert({ stripe_event_id, type }, { onConflict: 'stripe_event_id', ignoreDuplicates: true })`
3. `supabase.from('billing_events').select('processed_at').eq('stripe_event_id', event.id).maybeSingle()` — if `record?.processed_at` is truthy, returns `{ received: true, duplicate: true }` immediately (200), WITHOUT calling `handleEvent`.
4. On non-duplicate: `handleEvent(event)` is called if `SUPPORTED_EVENTS.has(event.type)`.
5. `supabase.from('billing_events').update({ processed_at: <iso> }).eq('stripe_event_id', event.id)`
6. Returns `{ received: true }` (200). On thrown error from step 4, returns 500 `'Processing error'` and does NOT run step 5 (so `processed_at` stays null — Stripe retries).

`handleEvent` branches (event.type):
- `checkout.session.completed`, `session.mode === 'subscription'` + `session.subscription` truthy: calls `stripe.subscriptions.retrieve(subId)` then `syncSubscription(subscription)`.
- `checkout.session.completed`, `session.mode === 'payment'` + `session.metadata?.kind === 'copilot_topup'`: calls `grantCopilot(orgId, pkg.creditsUsd, 'topup', ref, note)` where `pkg = CREDIT_TOPUP_PACKAGES[session.metadata.package]`. If org/package metadata missing, logs a warning and does nothing (no throw).
- `customer.subscription.created|updated|deleted`: calls `syncSubscription(event.data.object as Stripe.Subscription)`.
- `invoice.paid` / `invoice.payment_failed`: resolves `subId` via `invoiceSubscriptionId(invoice)` (checks `invoice.parent.subscription_details.subscription` first, falls back to legacy `invoice.subscription`). If found: `stripe.subscriptions.retrieve(subId)` then `syncSubscription(subscription)`. On `invoice.paid` ONLY, additionally calls `refreshCopilotAllowance(subscription)` which resolves org via `resolveOrgIdByCustomer(customerId)` (fallback `subscription.metadata.org_id`), resolves plan via `planByPriceId(item.price.id)`, and calls `resetCopilotForPeriod(orgId, plan.copilotIncludedUsd, periodEnd)`. No-ops silently if org or plan can't be resolved.

Mocked function imports the route depends on (mock all of these at module boundary):
```typescript
import { syncSubscription } from '@/lib/billing/sync'
import { resolveOrgIdByCustomer } from '@/lib/billing/customers'

```
`getStripe` from `@/lib/billing/stripe` must ALSO be mocked (Pitfall 3) so `stripe.subscriptions.retrieve(...)` returns a canned `Stripe.Subscription`-shaped object — but keep a SEPARATE, un-mocked, locally-constructed `new Stripe('sk_test_dummy')` instance in the test file purely for `generateTestHeaderString`/signing. These are two different Stripe instances: the mocked `getStripe()` the route calls, vs. the real one the TEST uses to sign payloads.

`CREDIT_TOPUP_PACKAGES` is imported directly from `@/lib/billing/catalog` (real, not mocked — it's a static object, no IO):
