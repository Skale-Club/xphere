## Context Override: Forced Dark Mode

This route group is **forced dark mode** regardless of system preference. Apply `class="dark"` on the `(admin)` layout `<html>` or wrapping element. The admin panel must visually differ from the dashboard: slate/zinc dark surfaces + red/orange accent instead of the dashboard's indigo accent.

Source: CONTEXT.md — "dark mode forced", "paleta: slate/zinc dark com accent vermelho/laranja"

## Typography

All sizes from the established project type scale. No new sizes introduced.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 regular | 1.5 |
| Label / table cell | 13px (0.8125rem) | 400 regular | 1.4 |
| Heading (page title, section) | 20px (1.25rem) | 600 semibold | 1.2 |
| Metric / display number | 28px (1.75rem) | 600 semibold | 1.1 |

Letter-spacing: `-0.015em` on headings, `-0.005em` on body (project standard from `globals.css`).
Tabular numerals (`font-variant-numeric: tabular-nums`) on all metric counts and ID columns — use `.tabular` utility class.

**Note on 13px vs 14px:** The 1px difference between label (13px) and body (14px) is intentional. It creates a clear label-vs-body hierarchy at small scale without introducing a third weight. Executors must not round 13px up to 14px.

## Color

This phase uses an **admin-specific color set** layered on top of the existing dark tokens. The forced dark palette is:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A0A0B` (`--bg-primary` dark) | Main page background behind sidebar and content area |
| Secondary (30%) | `#111113` (`--bg-secondary` dark) | Admin sidebar background, stat cards, table rows |
| Admin Accent (10%) | `#EF4444` / `#F97316` gradient | See reserved list below |
| Destructive | `#EF4444` (`--danger` dark) | Destructive actions only (feature flag disable confirmation) |

### Admin Accent Color Contract

The admin accent is a **red-to-orange gradient** (`from-red-500 to-orange-500`) used exclusively on:

1. The "SUPER ADMIN" badge in the header
2. Active sidebar nav item indicator (left border or background tint)
3. The sidebar logo/brand mark area background tint (`rgba(239, 68, 68, 0.08)`)
4. Primary action button ("Save Feature Flags" on feature flags form) — use `bg-red-600 hover:bg-red-700`

**Note on primary CTA vs destructive:** The primary CTA ("Save Feature Flags") uses `bg-red-600` (one stop darker than the destructive `red-500`). This is intentional: red-600 signals "privileged admin action" while red-500 is reserved for destructive/danger semantics. The two roles must never share the same shade.

The admin accent is **NOT used** on: table rows, metric numbers, regular text, links, or any non-privileged UI element.

### Surface Hierarchy (admin panel)

```

Sidebar:          #111113  (--bg-secondary dark)  — left 240px fixed

Table row hover:  #1A1A1D
Borders:          #2A2A2F  (--border dark)
Subtle border:    #1E1E22  (--border-subtle dark)
```

Text hierarchy:
```
Primary text:   #FAFAFA  (--text-primary dark)
Secondary text: #A1A1AA  (--text-secondary dark)
Tertiary / meta: #71717A (--text-tertiary dark)
```

## Page-by-Page Interaction Contracts

### `/admin` (redirect to `/admin/orgs`)

No content rendered — immediate redirect.

### `/admin/orgs` — Organizations List

**Layout:** Full-width table inside content area with page header.

**Focal point:** The organizations table is the primary focal point; the page heading and search bar act as secondary anchors. All visual weight flows toward the table.

**Page header:**
- Heading: "Organizations" at 20px / semibold
- Subtext: "All tenant organizations on the platform" at 13px / text-secondary

**Table columns:** Name | Slug | Created | Members | Contacts | Calls | Conversations | Actions

**Row interaction:**
- Hover: `bg-[#1A1A1D]` row highlight, transition 100ms
- Click row or "View" chevron → navigate to `/admin/orgs/[orgId]`
- No inline editing on this page

**Sort:** Column headers for Name and Created are sortable (client-side). Sorted column shows `ChevronUp` / `ChevronDown` icon at 14px. Default sort: Created descending (newest first).

**Search:** `Input` with `Search` icon prefix, placeholder "Search organizations…", filters table client-side by name or slug. Width: 280px, positioned right-aligned in page header row.

**Empty state:** See Copywriting Contract.

**Loading state:** 5 `Skeleton` rows, each 48px tall, column widths matching table layout.

### `/admin/orgs/[orgId]` — Organization Detail

**Layout:** Two-column on desktop (`grid grid-cols-3 gap-6`): main content (col-span-2) + sidebar panel (col-span-1).

**Metric cards:** 4 cards in a `grid grid-cols-2 gap-4` at top of main column.
- Each card: `Card` component, `CardContent p-4`
- Metric number: 28px / semibold / tabular — `.tabular` class
- Metric label: 13px / regular / text-secondary
- No trend indicators (static data, no real-time per CONTEXT.md)

**Members table:** Simple table below metrics — columns: Email | Role | Joined

**Feature flags panel:** Right sidebar column using `Card`.
- Section heading: "Feature Flags" 14px / semibold
- Each flag: `Switch` + label (14px regular) + description (12px text-tertiary) in a `flex items-center justify-between` row, **48px minimum height** (`2xl` token)
- Admin notes: `Input` below each flag, placeholder "Admin note…", 13px, only shown when flag is enabled (conditional render)
- Save button: "Save Feature Flags" — `bg-red-600 hover:bg-red-700 text-white`, full width, bottom of card

**Back navigation:** `Button variant="ghost"` with `ArrowLeft` icon, label "Back to Organizations", positioned above page heading.

### Feature Flags Defined

| Flag key | Label | Description |
|----------|-------|-------------|
| `ai_calling_enabled` | AI Calling | Enables outbound AI voice campaigns for this org |
| `bulk_import_enabled` | Bulk Import | Allows CSV bulk import of contacts |
| `advanced_pipeline_enabled` | Advanced Pipeline | Unlocks pipeline automation and multi-stage rules |

Source: CONTEXT.md — "Claude's Discretion" for flag naming; specific flags from `<specifics>` section.

## Accessibility Contract

- All `Switch` components: explicit `id` + `htmlFor` label association; `aria-describedby` pointing to description text
- Table: `<caption>` visually hidden but present for screen readers
- Admin badge: `role="status"` or rendered as `<span>` (not heading-level element)
- Focus ring: 2px `var(--accent)` outline (project standard, applies even in admin panel — indigo focus ring is acceptable as it is a system-wide accessibility affordance, not a brand accent)
- Color contrast: All text on dark surfaces meets WCAG AA (verified against existing dark tokens in `globals.css`)
- Keyboard: Table rows are not keyboard-navigable directly; "View" button/link in each row provides keyboard access
