136-01 / 136-02 exports the route composes:
```typescript
export const commerceEventSchema: ZodType   // .parse(JSON.parse(rawText)) → CommerceEventPayload (throws ZodError → 422)
export type CommerceEventPayload = { event_id: string; type: 'order.placed'|'customer.created'; occurred_at: string; data: {...} }
export type CommerceReceiptResult = { receiptId: string; duplicate: false } | { duplicate: true }
export async function insertCommerceReceipt(supabase, orgId, payload): Promise<CommerceReceiptResult>
export async function emitCommerceEvent(supabase, orgId, receiptId, type, data): Promise<{ dispatched:number; dispatchId:string|null }>
```

Route order of operations (136-RESEARCH Pattern 2 — each maps to a contract response):
```
1. content-length > 64*1024 → 413 { code:'payload_too_large' }              (extra-contractual guard, mirrors leads — Pitfall 6)
2. verifyApiKey(request, supabase, 'commerce:events') → !ok ⇒ 401 or 403 (auth.status)
3. R12: rateLimit('commerce:evt:'+auth.key.orgId, 600, 60, {failMode:'open'}) → !allowed ⇒ 429   (AFTER auth; keyed by org)
4. rawText = await request.text(); re-check byte length → 413; commerceEventSchema.parse(JSON.parse(rawText)) → ZodError ⇒ 422
5. idempotencyKey = header('idempotency-key')?.trim(); !idempotencyKey || idempotencyKey !== payload.event_id ⇒ 422
6. result = await insertCommerceReceipt(supabase, orgId, payload)
7. if (!result.duplicate) await emitCommerceEvent(supabase, orgId, result.receiptId, payload.type, payload.data)
8. await supabase.from('api_keys').update({ last_used_at: new Date().toISOString() }).eq('id', auth.key.keyId)
9. return result.duplicate
     ? Response.json({ duplicate: true }, { status: 200 })
     : Response.json({ receipt_id: result.receiptId }, { status: 201 })
export const runtime = 'nodejs'
```
