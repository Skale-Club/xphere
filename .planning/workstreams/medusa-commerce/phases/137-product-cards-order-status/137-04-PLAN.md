<objective>
When a verified commerce context carries an `email` claim, link the webchat conversation to a CRM contact: find-or-create the contact by normalised email, then set `conversations.contact_id` (only if currently null) and `visitor_email`. This happens at pin time in the chat route, right after `writeCommerceContext`, inside the existing fail-soft `try/catch` so it can never break the chat. It is throttled (already-linked conversations are skipped) and org-scoped.
