<objective>
Build the two Medusa cart-write executors and the thin primitives they compose:
the adoption-sig signer, a cart-only re-pin merge, a per-conversation write counter,
the fail-closed write budgets, and the shared guardrail/idempotency sets. Everything
here is unit-testable with mocked fetch + mocked rate-limit — no live store, no run-agent.

Purpose: This is the security-critical core of Phase 134. The `cart_created` sig must
byte-match stuscle's `verifyCartSig` or cart adoption silently fails; the create -> sign ->
pin -> emit ORDER is load-bearing; the clamps and fail-closed budgets are the anti-abuse
barrier. Getting these right in Wave 1 lets Wave 2 do pure wiring.

Output: `cart-sig.ts`, `add-to-cart.ts`, `update-cart-item.ts` (new); `context.ts`,
`client.ts`, `idempotency.ts`, `guardrails.ts` (extended); `tests/medusa-cart-write.test.ts`

Medusa 2.17.2 store cart WRITE endpoints (VERIFIED — pk header only, no auth):
| Op | Method + Path | Body | Response |
|----|---------------|------|----------|
| Create cart | POST /store/carts | { region_id } | { cart } |
| Set metadata | POST /store/carts/:id | { metadata: { xphere_sig } } | { cart } |
| Add line item | POST /store/carts/:id/line-items | { variant_id, quantity } | { cart } (FULL, with items) |
| Update line item | POST /store/carts/:id/line-items/:line_id | { quantity } | { cart } |
| Delete line item | DELETE /store/carts/:id/line-items/:line_id | — | { deleted, parent: cart }  ← read .parent, NOT .cart |

Append `?fields=id,currency_code,+total,*items,+items.total,*items.variant` to add/update
calls so `items` (id, variant_id, title, quantity, total) + totals are present without a 2nd GET.
