  <action>
Read each of the five files listed above. For each file, search for patterns that would break light mode:

1. Hardcoded dark hex strings that are NOT inside a dark: Tailwind prefix and NOT inside a file that's intentionally dark-only:
   - Inline style={{ backgroundColor: '#0A0A0B' }} or similar dark hex values
   - className strings containing literal dark hex like bg-[#0A0A0B], text-[#FAFAFA]
   - Any color value that hardcodes dark-palette values without a light-mode counterpart

2. For each problematic pattern found:
   - If it uses a color that maps to a CSS variable (e.g., #0A0A0B = var(--bg-primary)), replace with the CSS variable or the Tailwind token (e.g., bg-bg-primary, text-text-primary)
   - If it uses a Tailwind color without a dark: alternative and will look wrong on light, add the appropriate dark: override or default value

3. If a file has NO hardcoded dark hex and only uses CSS variables or Tailwind tokens with proper dark: prefixes, no changes are needed — note it as "clean" in your summary.

Audit decision table:
- bg-[#0A0A0B]  → replace with bg-bg-primary  (maps to --bg-primary)
- bg-[#111113]  → replace with bg-bg-secondary (maps to --bg-secondary)
- bg-[#1A1A1D]  → replace with bg-bg-tertiary  (maps to --bg-tertiary)
- text-[#FAFAFA] → replace with text-text-primary (maps to --text-primary)
- text-[#A1A1AA] → replace with text-text-secondary (maps to --text-secondary)
- border-[#2A2A2F] → replace with border-border   (maps to --border)

Do NOT touch: className="dark" wrappers — these are scope locks, not color overrides.
Do NOT touch: the intentionally dark-only files listed in the interface context above.
