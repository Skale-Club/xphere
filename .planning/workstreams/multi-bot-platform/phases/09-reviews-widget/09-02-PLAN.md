  <action>
    Create `src/app/api/reviews/[token]/route.ts` as a Node.js route handler using `createServiceRoleClient()`.

    Behavior requirements:
    1. Support `OPTIONS` and `GET` with permissive CORS headers for public embed usage.
    2. Load one `google_locations` row by `review_token` and nested `google_reviews(*)` ordered by `display_order`.
    3. Reject with non-200 when:
       - token is invalid
       - location has zero reviews
       - `fetched_at` is null
       - `fetched_at` is older than 30 days
    4. Return compact JSON only from cached DB data; do not call Google Places API here.
    5. Include place name and maps URL so the widget can render attribution and optional CTA.
    6. Keep response lean and public-safe; no org IDs, no internal errors leaked.

    Recommended response behavior:
    - `OPTIONS` -> `204`
    - invalid/missing/stale -> `404` with `{ error: 'Unavailable' }`
    - valid -> `200` with `ReviewsWidgetPayload`
  </action>
  <acceptance_criteria>
    - route exists at `src/app/api/reviews/[token]/route.ts`
    - file exports `runtime = 'nodejs'`
    - file exports both `GET` and `OPTIONS`
    - route uses `createServiceRoleClient()`
    - route contains a 30-day stale check against `fetched_at`
  </acceptance_criteria>
  <verify>
    <automated>npx vitest run tests/reviews-widget-route.test.ts --reporter=verbose</automated>
  </verify>

    Create `src/reviews-widget/index.ts` as a standalone vanilla TypeScript IIFE source with no React imports.

    Required behavior:
    1. Read `data-token` from `document.currentScript` before any async work; abort silently when missing.
    2. Read optional embed attributes:
       - `data-layout` -> `carousel | grid | list | compact`
       - `data-theme` -> `light | dark`
       - `data-primary-color`
       - `data-star-color`
       - `data-show-photo`
       - `data-show-date`
       - `data-show-google-button`
       - `data-border-radius`
       - `data-max-width`
    3. Create one host root, attach Shadow DOM, inject inline CSS, and never mutate host-page layout beyond the host node.
    4. Fetch `/api/reviews/{token}` from the script origin with `Accept: application/json`.
    5. Render 4 layouts with one shared review-card renderer:
       - carousel: horizontal snap cards with previous/next buttons
       - grid: responsive two-column cards
       - list: vertical stacked cards
       - compact: tight single-column quote cards
    6. Always render:
       - star rating
       - author name adjacent to review text
       - `Powered by Google` attribution using `/google-logo.svg`
    7. Optionally render photo/date/button based on data attributes.
    8. On non-200, malformed JSON, or empty reviews, remove or hide the host root and swallow errors.

    Use CSS custom properties for appearance overrides so layout and theming stay inside the Shadow DOM.
  </action>
