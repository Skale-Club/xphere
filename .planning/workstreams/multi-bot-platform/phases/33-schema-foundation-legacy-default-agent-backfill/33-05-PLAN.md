<objective>
Write migration 039, which adds the additive nullable `agent_id` column to the two channel-config tables (`manychat_rules` and `meta_channels`) so that, in Phase 37, the inbound dispatcher can branch: if the rule/channel has an `agent_id`, route the message through `runAgent()`; otherwise, fall back to the legacy v1.x `tool_config_id` dispatch path.

Purpose: This is the structural foundation for CHAN-06 (multi-channel agent dispatch). Phase 33 does NOT switch any existing rule over — D-33-11 explicitly forbids backfill so existing manychat rules and meta channels continue to work byte-identically. The XOR CHECK constraint (only one of `agent_id` / `tool_config_id` may be set per row) is intentionally deferred to Phase 37 (D-33-12) because adding it now would block existing rows that have `tool_config_id` set and `agent_id` NULL — they'd pass the CHECK only by coincidence today, but tomorrow's Phase 36 admin UI needs the column writable.

Output: 1 migration file. Not pushed in this plan — Plan 07 runs `npx supabase db push`.
</objective>
