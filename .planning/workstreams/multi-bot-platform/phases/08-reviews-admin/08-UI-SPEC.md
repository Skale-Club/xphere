## Color

All values from `src/app/globals.css` CSS custom properties. Dark mode is supported via `.dark` class — use semantic tokens only, never raw hex.

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | Page background, card backgrounds |
| Secondary (30%) | `--card` / `--muted` | `hsl(240 4.8% 95.9%)` | Location cards, add-location form panel, skeleton backgrounds |
| Accent (10%) | `--primary` | `hsl(240 5.9% 10%)` | "Add Location" submit button, "Sync Reviews" button |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | "Delete location" button and its confirmation dialog |

Accent (`--primary`) reserved for:
1. "Add Location" primary submit button (form CTA)
2. "Sync Reviews" button when cooldown is NOT active

All other interactive elements (cancel, secondary links, cooldown-disabled sync) use `variant="outline"` or `variant="ghost"` — never accent color.

Error/warning surface: Use `bg-destructive/10 text-destructive border-destructive/20` for inline error display (last_fetch_error). Same pattern as existing `integrations-table.tsx` badge pattern.

Attribution badge ("Powered by Google"): `text-xs text-muted-foreground` with the official Google logo SVG served from `/public/google-logo.svg`. No custom color override.

Primary focal point: "Add Location" CTA button (accent color, top of content area).

### LocationCard Layout

Each registered location renders as a `rounded-md border` card (matches `integrations-table.tsx` `rounded-md border divide-y` pattern):

```
LocationCard (rounded-md border)
├── Card header row (px-4 py-3, flex justify-between items-start)
│   ├── LEFT: MapPin icon + name (text-sm font-normal) + client_name badge
│   │         address (text-xs text-muted-foreground)
│   │         category (text-xs text-muted-foreground, if present)
│   └── RIGHT: SyncButton + DeleteButton
│
├── Sync status row (px-4 py-2, border-t, text-xs text-muted-foreground)
│   ├── "Last synced: {formatDistanceToNow(fetched_at)} ago" OR "Never synced"
│   ├── "{review_count} review(s)"
│   └── Error badge (if last_fetch_error): AlertCircle icon + truncated error text
│
├── Separator (if reviews exist)
│
└── Review rows (max 5, divide-y inside)
    Each review: px-4 py-3, flex gap-3
    ├── Avatar (h-8 w-8, AvatarImage from author_photo_url, AvatarFallback initials)
    └── Review content column
        ├── Author name (text-sm font-normal) + star rating row (5 Star icons, 12px)
        ├── relative_time (text-xs text-muted-foreground)
        └── review_text (text-sm, line-clamp-3)
        └── "View on Google" link (text-xs text-primary underline, opens in new tab)
```

Google attribution must appear once per LocationCard that has reviews, immediately after the last review row:

```
<p className="px-4 py-2 text-xs text-muted-foreground flex items-center gap-1">
  <img src="/google-logo.svg" alt="Google" className="h-3 w-auto" />
  Powered by Google
</p>
```

### AddLocationForm Layout

Rendered as a `Card` (always visible, not collapsible) above the location list:

```
Card
├── CardHeader
│   ├── CardTitle: "Add a location"
│   └── CardDescription: "Enter the Google Place ID to register a new location."
└── CardContent
    └── <form> (space-y-4)
        ├── Row 1: Place ID (required) — full width Input
        ├── Row 2: Location name (required) — full width Input
        ├── Row 3: Client name (optional) — full width Input
        ├── Row 4: Address (optional) — full width Input
        ├── Row 5: Maps URL (optional) — full width Input
        ├── Row 6: Category (optional) — full width Input
        └── Row 7: flex gap-3
                   Button type="submit" (primary): "Add Location"
                   — shows Loader2 + "Adding..." while pending
```

Form collapsed to single row of required fields only if more than 3 locations already exist — no, keep always expanded. Simplicity over progressive disclosure for admin tooling.

## Interaction States

### SyncButton (`sync-button.tsx`, `'use client'`)

| State | Visual | Behavior |
|-------|--------|----------|
| Default (no prior sync) | Button variant="outline", RefreshCw icon, label "Sync Reviews" | Clickable; triggers `syncReviews` server action via `useTransition` |
| Loading (sync in progress) | Button disabled, `Loader2 animate-spin`, label "Syncing…" | Non-clickable; `isPending` from `useTransition` |
| Success | Button re-enabled; `sonner` toast: "Reviews synced — X review(s) loaded." | `revalidatePath` refreshes the card data |
| Cooldown active | Button disabled, variant="outline", RefreshCw icon (no spin), label "Sync Reviews" | Wrapped in `TooltipProvider`; tooltip shows "Available in Xh (24h minimum between syncs)" |
| Error (API or config) | Button re-enabled; `sonner` toast error; error stored in `last_fetch_error` and shown in card | Cooldown NOT activated if sync failed (no `fetched_at` update on error) |

Cooldown detection: computed client-side from `fetched_at` prop passed to `SyncButton`. If `fetched_at` is non-null and `Date.now() - new Date(fetched_at).getTime() < 24 * 3600 * 1000`, disable with tooltip.

### DeleteButton

| State | Visual | Behavior |
|-------|--------|----------|
| Default | Button variant="ghost" className="text-destructive hover:bg-destructive/10", Trash2 icon, no label (icon only with tooltip "Delete location") | Opens AlertDialog on click |
| AlertDialog open | Destructive confirmation dialog | Blocks background; "Cancel" and "Delete location" actions |
| Deleting | AlertDialog action button: Loader2 + "Deleting…", disabled | Triggers `deleteLocation` server action |

### AddLocationForm

| State | Visual | Behavior |
|-------|--------|----------|
| Idle | All fields enabled | — |
| Submitting | All fields disabled, submit button shows Loader2 + "Adding…" | `useTransition` or `useState` isPending |
| Validation error | `FormMessage` beneath the field | `zod` inline error, no toast |
| Server error | `sonner` toast error | e.g. "Failed to add location. Check the Place ID and try again." |
| Success | `sonner` toast: "Location added." Form resets to empty | `revalidatePath('/reviews')` refreshes list |

## Google Attribution Contract (ToS Requirement)

"Powered by Google" attribution MUST appear on every `LocationCard` that shows review data.

- Position: bottom of the review list within the card, before the card border ends
- Layout: `px-4 py-2 flex items-center gap-1 text-xs text-muted-foreground`
- Logo: `<img>` tag pointing to `/public/google-logo.svg`, `h-3 w-auto`, `alt="Google"`
- If zero reviews are fetched (empty `google_reviews` array), attribution is NOT shown (no reviews to attribute)
- Author names (`author_name`) MUST be visible adjacent to each review — never hide or truncate author attribution

## Star Rating Display

- 5 `Star` icons from lucide-react rendered inline
- Filled stars: `fill-current text-yellow-400` (using Tailwind `yellow-400` — the one exception to semantic-only colors, matching universal star rating convention)
- Empty stars: `text-muted` (unfilled, no fill)
- Icon size: `h-3 w-3`
- Gap between stars: `gap-1` (4px — stars at 12px fit cleanly with 4px gap)
- Rating is integer 1–5 from DB; no half-star display

## Accessibility

- All icon-only buttons (delete) must include a `Tooltip` with a text label and `aria-label` attribute on the button element.
- `SyncButton` disabled state must include `aria-disabled="true"` and `aria-describedby` pointing to the cooldown tooltip content.
- Star rating rows must include `aria-label="{N} out of 5 stars"` on their container span.
- "View on Google" links must include `target="_blank" rel="noopener noreferrer"` and `aria-label="View {author_name}'s review on Google (opens in new tab)"`.
