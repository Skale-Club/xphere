  <action>
    Create `src/app/api/meta/callback/route.ts` as a Node.js GET route.

    Required behavior:
    1. Validate the current admin session with `getUser()`; redirect unauthenticated users to `/login`.
    2. Read `code` and `state` from the query string; reject missing or mismatched state by redirecting to `/integrations/meta?error=csrf` or `/integrations/meta?error=missing_code`.
    3. Run the full token exchange chain:
       - authorization code -> short-lived user token
       - short-lived user token -> long-lived user token
       - long-lived user token -> `/me/accounts` page tokens
       - page lookup -> linked `instagram_business_account` when present
    4. Encrypt each Page Access Token with `encrypt()` before writing to `meta_channels`.
    5. Upsert one `messenger` row per page and one `instagram` row when an IG account exists, clearing `connection_error`, setting `is_active = true`, and stamping `last_synced_at`.
    6. Redirect success back to `/integrations/meta?connected=true`; redirect exchange failures to `/integrations/meta?error=oauth_exchange`.

    Important:
    - `export const runtime = 'nodejs'`
    - Do not persist short-lived or long-lived user tokens.
    - Do not add a new migration.
  </action>

    - callback redirects to `/integrations/meta` for both success and handled errors
