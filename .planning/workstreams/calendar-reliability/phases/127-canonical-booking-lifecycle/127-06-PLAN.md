<objective>
Fix `src/lib/flows/engine.ts`'s six booking-mutating action-node handlers (`executeBookingConfirm`, `executeBookingCancel`, `executeBookingReschedule`, `executeBookingMarkNoShow`, `executeBookingMarkComplete`, `executeBookingCreate`) — the durable/wait-containing workflow engine's booking actions. RESEARCH confirmed none of these six call `emitCalendarEvent` at all (a workflow that confirms/cancels/reschedules/no-shows/completes a booking today produces zero downstream calendar-workflow effects), none check that the target `booking_id` belongs to the running workflow's own `org_id` (a workflow config referencing another org's `booking_id` can mutate it today with no boundary check at all), and `executeBookingMarkComplete` writes the literal, DB-CHECK-violating value `status: 'completed' as 'confirmed'` — a live crash every time this action node actually executes against a real database, using a local `BookingStatus` type (`'confirmed' | 'cancelled' | 'no_show' | 'pending' | 'completed'`) that disagrees with the DB on two counts (missing `showed`, includes two non-existent values).

This plan replaces the five status-transition handlers' inline guard+update logic with delegation to Plan 127-01's canonical `transition.ts` functions (closing all three gaps above at once — event emission, org-scoping, and valid status vocabulary), and adds `meeting.scheduled` event emission to `executeBookingCreate` for consistency with every other booking-creation writer in this codebase. It also closes the loop on `tests/calendar-status-vocabulary.test.ts` (Plan 127-01): that test deliberately excluded `src/lib/flows/engine.ts` from its scanned-files list because this file still contained the invalid `'completed'` literal at the time 127-01 ran; once this plan removes that literal, this plan adds the file back to the scan list so the static vocabulary guard covers it going forward.

Purpose: closes the largest concrete LIFE-03 gap RESEARCH found (an entire writer category emitting nothing) and the literal live-crashing bug behind LIFE-02.
Output: all six `executeBooking*` handlers in `src/lib/flows/engine.ts` delegate to the canonical service; the file's own mismatched local `BookingStatus` type is removed; `tests/calendar-status-vocabulary.test.ts` now scans `src/lib/flows/engine.ts` too.
</objective>
