From src/app/(dashboard)/calendar/_actions/bookings.ts (lines 92-108):
```typescript
async function resolveHostName(userId: string): Promise<string> {
  try {
    const svc = createServiceRoleClient()
    const { data } = await svc.auth.admin.getUserById(userId)
    const user = data?.user
    if (!user) return 'your host'
    const meta = (user.user_metadata ?? {}) as Record<string, unknown>
    const fullName =
      (typeof meta.full_name === 'string' && meta.full_name) ||
      (typeof meta.name === 'string' && meta.name) ||
      null
    if (fullName) return fullName
    return user.email ?? 'your host'
  } catch {
    return 'your host'
  }
}
```
`auth.admin.getUserById` is a service-role-only Supabase Admin API method —
it will fail silently/throw if called on a non-service-role client. Use
`createServiceRoleClient()` from `@/lib/supabase/admin` inside the
organizer-hydration fix regardless of what client `buildMeetingScope` itself
receives as a parameter (Pitfall 3 in 130-RESEARCH.md).

`event_types` has NO `name` column — only `title` (verified against
`src/types/database.ts`'s `event_types.Row` type and
`src/app/(dashboard)/calendar/_actions/event-types.ts::EventTypeRow`). Any
`.select('... name ...')` against `event_types` is a join bug; the correct
form is `.select('... name:title ...')` (alias), which keeps the rest of the
consuming code's `.name` reads correct without renaming them.
