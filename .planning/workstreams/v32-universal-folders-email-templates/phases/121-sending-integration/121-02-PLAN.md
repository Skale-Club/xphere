MECHANISM (how a tool becomes real, end to end):
1. A workflow node's `kind` in YAML IS the action_type passed to executeAction
   (see run-flow-sync.ts normalizeGraph: "node.kind is the action_type for action nodes").
2. The validator (src/lib/workflows/validate.ts) accepts a node kind iff it exists in
   `spec.nodes` — i.e. the `NODES` array in src/lib/workflows/spec.ts.
3. The CLI `npm run workflows:validate` (scripts/validate-workflows.ts) builds a STATIC
   spec from `NODES` and validates every .yaml under supabase/seeds/workflows/ (recursive).
4. getWorkflowSpec() org-filters NODES: a node is dropped unless its `integration_required`
   list is empty OR at least one required provider is connected for the org. So setting
   `integration_required: ['resend']` makes send_email_template appear ONLY when email is
   configured (the org-gate UFE-11 asks for).

The NodeSpec shape (from spec.ts):
```typescript
export interface NodeSpec {
  type: string
  kind: 'action' | 'condition' | 'wait' | 'agent' | 'end'
  description: string
  integration_required?: string[]   // matches integrations.provider
  params_schema?: Record<string, unknown>
  examples?: Array<Record<string, unknown>>
}
```

The EXISTING send_email NodeSpec (spec.ts ~line 809, "Action | email" section) — the sibling to mirror:
```typescript
{
  type: 'send_email',
  kind: 'action',
  description: 'Send a transactional email using the platform Resend settings.',
  params_schema: {
    type: 'object',
    properties: {
      to: { type: 'string', description: 'Recipient email address.' },
      subject: { type: 'string' },
      body: { type: 'string', description: 'HTML email body.' },
    },
    required: ['to', 'subject', 'body'],
  },
},
```

tool_call seed shape (from .planning/workflows/examples/tool-send-appointment-sms.yaml):
trigger.type: tool_call with config.tool_name + config.input_schema (one entry per field,
each { type, description, required }). The validator REQUIRES a non-empty input_schema for
tool_call triggers.

Valid provider slugs (integration_provider enum): includes 'resend'. Use 'resend'.
