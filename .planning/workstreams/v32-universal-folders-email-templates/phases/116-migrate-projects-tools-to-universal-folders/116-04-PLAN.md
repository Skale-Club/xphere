<objective>
Swap the Tools module's folder operations off the legacy `tool_folders` table and onto
the universal `folders` store (entity_type='tool'), delegating folder CRUD to the
shared foldering core. Tools' item column is already `folder_id` (the core default), so
no `itemFolderColumn` override is needed.

Purpose: Second half of Phase 116 for Tools. The 1227 migration (plan 116-02) copies
`tool_folders` into `folders` preserving UUIDs and repoints `tool_configs.folder_id`
-> `folders(id)`; this plan rewires the CODE. The Tools folder actions live in
`src/app/(dashboard)/workflows/actions.ts` (Tools are surfaced under the Workflows
section — hence `revalidatePath('/workflows')`), NOT under a `tools/` directory. A
second consumer, `src/app/(dashboard)/agents/_actions/tools.ts` (the agent tool picker),
also queries `tool_folders` and must swap.

CRITICAL CONTRACT — the Tools consumers expect the OLD return shapes, NOT the core's
`ActionResult<T>`. `src/components/tools/tools-table.tsx` imports the mutation actions
(`createFolder`, `updateFolder`, `deleteFolder`, `deleteFolderWithTools`,
`reorderFolders`, `moveToolToFolder`) and the `ToolFolder` type; it branches on
`result.error` (i.e. expects `{ error?: string } | void`). `getFolders()` returns a
BARE `ToolFolder[]` (not wrapped). So the delegations must ADAPT the core's
`ActionResult<T>` back to these legacy shapes. This is different from the projects
wrapper (116-03), which keeps ActionResult.

Output:
- `workflows/actions.ts`: the tool-FOLDER functions (`getFolders`, `createFolder`,
  `updateFolder`, `deleteFolder`, `deleteFolderWithTools`, `reorderFolders`,
  `moveToolToFolder`) delegate to `@/lib/foldering/core` (entity_type='tool',
  itemTable='tool_configs') while preserving their legacy return shapes. The
  `ToolFolder` type stays exported and assignable from a folders Row.
- `agents/_actions/tools.ts`: the `.from('tool_folders')` picker query swaps to
  `.from('folders').eq('entity_type','tool')`, and the `folders` prop type swaps off
  `Database['public']['Tables']['tool_folders']['Row']`.

Depends on 116-01 (imports FolderingContext; default folder_id path only, but shares
the core module contract). Independent of 116-03.

Scope guardrails (do NOT touch):
- The `_legacy_tool_configs` queries in `workflows/actions.ts` (`getToolConfigs`,
  `renameToolConfig`, `deleteToolConfig`, `moveToolToFolder`'s tool-side update) target
  the TOOL CONFIGS table, not `tool_folders`. They are NOT part of this migration.

  unchanged by the migration, so its `.from('_legacy_tool_configs').update({ folder_id })`
  stays as-is (or optionally delegates to core.moveItemToFolder with itemTable
  '_legacy_tool_configs'; keep it simple and leave the direct update unless a cleaner

   overkill for a single row). Simplest faithful mapping:
```typescript
export async function updateFolder(
  id: string,
  data: { name?: string; position?: number },
): Promise<{ error?: string } | void> {
  const user = await getUser()
  if (!user) return { error: 'Not authenticated.' }
  const ctx = await toolCtx()
  if (data.name !== undefined) {
    const res = await core.renameFolder(ctx, id, { name: data.name })
    if (!res.ok) return { error: res.error }
  }
  if (data.position !== undefined) {
    const { error } = await ctx.supabase
      .from('folders')
      .update({ position: data.position })
      .eq('id', id)
      .eq('entity_type', 'tool')
    if (error) return { error: error.message }
  }
  revalidatePath('/workflows')
}
```

export async function deleteFolder(id: string): Promise<{ error?: string } | void> {
  const user = await getUser()

}

5. `deleteFolderWithTools(id)` — old collects subfolder ids then deletes the folder

   `folders` and delete from `folders`:
```typescript
export async function deleteFolderWithTools(id: string): Promise<{ error?: string } | void> {
  const user = await getUser()
  if (!user) return { error: 'Not authenticated.' }
  const supabase = await createClient()
  // Subfolder collection kept for parity with prior behavior; CASCADE handles the rest.
  await supabase.from('folders').select('id').eq('entity_type', 'tool').eq('parent_id', id)
  const { error: folderError } = await supabase
    .from('folders')
    .delete()
    .eq('id', id)
    .eq('entity_type', 'tool')
  if (folderError) return { error: folderError.message }
  revalidatePath('/workflows')
}
```
