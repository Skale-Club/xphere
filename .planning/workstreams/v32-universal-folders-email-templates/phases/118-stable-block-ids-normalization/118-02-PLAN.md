<objective>
Refactor the email-template editor from index-path block addressing to stable
id-based addressing, and wire in the id-minting introduced in Plan 118-01.

This is the UI half of Phase 118. After Plan 118-01, every block has an `id` in the
data model and `normalizeDocument` backfills legacy docs. This plan makes the editor
USE those ids: selection state becomes `selectedBlockId`, add/remove/update/insert
locate blocks by id, block lists key by `block.id`, block creation mints a fresh id,
and reusable-block insert re-mints ids so inserting the same block twice never
collides.

Purpose: Index-keyed lists break under reordering/insertion (the exact operations
Phase 119 drag-and-drop introduces). Stable id keys are the prerequisite. This phase
ships ZERO user-visible change — it is pure plumbing.

Output: A refactored `email-template-editor.tsx` that compiles, renders existing
templates identically, and keys everything by id.
</objective>
