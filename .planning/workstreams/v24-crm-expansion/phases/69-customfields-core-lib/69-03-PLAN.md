<feature>
  <name>Custom Fields Validator + Serializer unit tests</name>
  <files>tests/customfields-validator.test.ts</files>
  <behavior>
    Six test groups required (map directly to locked design decision §6):

    1. Unknown key rejection
       - values = { unknown_key: 'x' }, definitions = [] → { ok: false, errors: [{ field: 'unknown_key', message: 'unknown_custom_field' }] }
       - values = { known: 'hi', unknown_key: 'x' }, definitions = [{key:'known', type:'text', required:false, unique_per_org:false}] → error for unknown_key only

    2. Required enforcement
       - definition with required=true, value absent from payload → { ok: false, errors: [{ field: ..., message: 'required' }] }
       - definition with required=true, value present → { ok: true }
       - definition with required=false, value absent → { ok: true }

    3. Type coercion (number, boolean, date)
       - type='number', value='not-a-number' → { ok: false, errors: [{ field: ..., message: 'invalid_type' }] }
       - type='boolean', value=true → { ok: true }
       - type='date', value='2024-01-15' → { ok: true }
       - type='date', value='not-a-date' → { ok: false, errors }

    4. Currency round-trip (CF-15)
       - parseCurrencyValue("1500 BRL") → { amount: 1500, currency: "BRL" }
       - parseCurrencyValue({ amount: 1500, currency: "BRL" }) → { amount: 1500, currency: "BRL" } (pass-through)
       - parseCurrencyValue("invalid") → throws
       - normalizeCustomFieldValues with type='currency', value="2000 USD" produces { amount: 2000, currency: "USD" }
       - validateCustomFields with type='currency', value={ amount: 100, currency: 'USD' } → { ok: true }

    5. unique_per_org (mocked DB returns existing row)
       - Mock supabase query returns [{id: 'existing'}] for the entity table → { ok: false, errors: [{ field: ..., message: 'unique_per_org' }] }
       - Mock supabase query returns [] → { ok: true }

    6. Serializer tests (pure functions, no mocking needed)
       - normalizeCustomFieldValues coerces number-string to number for type='number'
       - normalizeCustomFieldValues coerces "true" to true for type='boolean'? (check actual implementation — may stay as string and let zod reject it)
       - normalizeCustomFieldValues returns new object without mutating input
       - normalizeCustomFieldValues for multi_select: "a,b,c" → ["a","b","c"]
  </behavior>
