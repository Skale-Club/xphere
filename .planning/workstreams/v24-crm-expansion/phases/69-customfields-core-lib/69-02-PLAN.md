  <behavior>
    - validateCustomFields(orgId, 'contact', {}) returns { ok: true } when custom_field_definitions table is empty for that org+entity
    - validateCustomFields with a key not present in definitions returns { ok: false, errors: [{ field: key, message: 'unknown_custom_field' }] }
    - validateCustomFields with a required definition and missing key returns { ok: false, errors: [{ field: key, message: 'required' }] }
    - validateCustomFields collects ALL errors (not fail-fast) — required error + unknown key error both appear in a single response
    - validateCustomFields for unique_per_org=true: queries the entity table to check for existing rows with the same value; if found returns { ok: false, errors: [{ field: key, message: 'unique_per_org' }] }
    - validateCustomFields for type 'number' with value "not-a-number" returns { ok: false, errors: [{ field: key, message: 'invalid_type' }] }
    - validateCustomFields for type 'currency' with { amount: 100, currency: 'USD' } returns { ok: true }
    - validateCustomFields only queries definitions WHERE archived = false
    - validateCustomFields runs identically for entity = 'contact' | 'opportunity' | 'account'
  </behavior>
  <action>
    Create src/lib/custom-fields/validate.ts:

    1. Imports:
       ```typescript
       import { createClient } from '@/lib/supabase/server'
       import type { CustomFieldEntity } from '@/types/database'
       import { FIELD_RENDER_CONFIG } from './render-config'
       import { parseCurrencyValue } from './serialize'
       ```

    2. Export FieldError interface: { field: string; message: string }
    3. Export FieldValidationResult type: { ok: true } | { ok: false; errors: FieldError[] }

    4. Export async function validateCustomFields(
         orgId: string,
         entity: CustomFieldEntity,
         values: Record<string, unknown>
       ): Promise<FieldValidationResult>

       Implementation steps inside the function:

       a. Load definitions:
          ```typescript
          const supabase = await createClient()
          const { data: definitions, error: dbErr } = await supabase
            .from('custom_field_definitions')
            .select('id, key, type, required, unique_per_org, options, validation')
            .eq('org_id', orgId)
            .eq('entity', entity)
            .eq('archived', false)
          if (dbErr) return { ok: false, errors: [{ field: '_db', message: dbErr.message }] }
          ```

       b. Build a lookup map: defsByKey = new Map(definitions.map(d => [d.key, d]))

       c. Collect errors (errors: FieldError[] = [])

       d. Unknown key check — for each key in Object.keys(values): if !defsByKey.has(key), push { field: key, message: 'unknown_custom_field' }

       e. For each definition in definitions:
          - Required check: if definition.required && !(key in values) || values[key] == null || values[key] === '', push { field: def.key, message: 'required' }
          - Skip further checks for missing optional fields
          - Type check: use FIELD_RENDER_CONFIG[def.type].zodSchema.safeParse(values[def.key]); if !result.success push { field: def.key, message: 'invalid_type' }
          - For currency type: wrap parseCurrencyValue(values[def.key]) in try/catch; on throw push { field: def.key, message: 'invalid_currency_value' }
          - unique_per_org check (only when value is present and type check passed):
            if (def.unique_per_org) {
              const table = entity === 'contact' ? 'contacts' : entity === 'opportunity' ? 'opportunities' : 'accounts'
              const { data: existing } = await supabase.from(table).select('id').filter(`custom_fields->>${def.key}`, 'eq', String(values[def.key])).limit(1)
              if (existing && existing.length > 0) push { field: def.key, message: 'unique_per_org' }
            }

       f. Return errors.length === 0 ? { ok: true } : { ok: false, errors }

    Note: The unique_per_org query uses JSONB ->> operator for text extraction. This is accurate for text/number/boolean scalar types. Currency values stored as nested JSON won't match via ->> on the full key (acceptable in v1 — currency fields are unlikely to be unique_per_org).

    5. Update src/lib/custom-fields/index.ts to add: export * from './validate'
