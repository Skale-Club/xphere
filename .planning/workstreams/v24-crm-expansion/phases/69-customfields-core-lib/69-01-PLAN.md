    3. Export FIELD_RENDER_CONFIG: Record<CustomFieldType, FieldRenderConfig> covering all 13 types:
       - text: inputType='text', zodSchema=z.string(), displayFormatter=String

       - integer: inputType='number', zodSchema=z.number().int(), displayFormatter=(v)=>String(v)
       - boolean: inputType='checkbox', zodSchema=z.boolean(), displayFormatter=(v)=>v?'Yes':'No'
       - date: inputType='date', zodSchema=z.string().regex(/^\d{4}-\d{2}-\d{2}$/), displayFormatter=String
       - datetime: inputType='datetime-local', zodSchema=z.string().datetime(), displayFormatter=String
       - select: inputType='select', zodSchema=z.string(), displayFormatter=String
       - multi_select: inputType='multi-select', zodSchema=z.array(z.string()), displayFormatter=(v)=>Array.isArray(v)?v.join(', '):String(v)
       - url: inputType='url', zodSchema=z.string().url(), displayFormatter=String
       - email: inputType='email', zodSchema=z.string().email(), displayFormatter=String
       - phone: inputType='tel', zodSchema=z.string().regex(/^\+[1-9]\d{1,14}$/), displayFormatter=String
       - currency: inputType='currency', zodSchema=z.object({ amount: z.number(), currency: z.string().length(3) }), displayFormatter=(v)=>{ const c=v as {amount:number;currency:string}; return `${c.currency} ${c.amount}` }

    No React imports. No 'use client'. This file is read-only config for Phase 71's renderer.

    Create src/lib/custom-fields/serialize.ts:

    1. Import FIELD_RENDER_CONFIG from './render-config' and CustomFieldType from '@/types/database'.
    2. Define CurrencyValue interface: { amount: number; currency: string }.
    3. Export parseCurrencyValue(raw: unknown): CurrencyValue — handles:
       - Object with amount + currency properties → validate and return
       - String "1500 BRL" (number space 3-letter-code) → parse and return
       - Otherwise → throw new Error('invalid_currency_value')
    4. Export normalizeCustomFieldValues(values: Record<string, unknown>, definitions: Array<{ key: string; type: CustomFieldType }>): Record<string, unknown>:
       - Iterates entries; for each key finds matching definition by key
       - If no definition found: keeps value as-is (unknown key rejection happens in validate.ts, not here)
       - Applies type coercion per type using FIELD_RENDER_CONFIG[def.type].zodSchema.safeParse(); on parse failure, keeps original value (validate.ts rejects it with a typed error)
       - Special cases: 'currency' → parseCurrencyValue; 'number'/'integer' → Number(val); 'boolean' → Boolean coercion; 'multi_select' → string split on comma if string
       - Returns new object — does not mutate input
  </action>
