<objective>
Create migration `supabase/migrations/064_accounts.sql` that introduces the `accounts` table as a first-class CRM entity, adds nullable `account_id` FK columns on `contacts` and `opportunities`, enforces a CHECK constraint that every opportunity has at least one of `contact_id` or `account_id`, and idempotently migrates existing `contacts.company` free-text values into `accounts` rows. The migration must be safe to run twice with no duplicate side effects, must be Hetzner-portable (pure Postgres, no Vercel-specific constructs), and must apply cleanly via `npx supabase db push`.

Purpose: This migration is the foundation for the entire v2.4 CRM Expansion milestone (phases 64–75). Without `accounts` existing first, the custom-fields system (Phase 68) cannot include `account` in its entity enum and the import pipeline (Phase 75) cannot auto-link `contact.account_id`.

CREATE TABLE IF NOT EXISTS public.contacts (

  created_at timestamptz NOT NULL DEFAULT now(),

**Section 8 — CHECK constraint on `opportunities`:**
The constraint must be added idempotently — Postgres does not support `ADD CONSTRAINT IF NOT EXISTS`, so guard with a DO block that probes `pg_constraint`.
```sql
DO $$
BEGIN
  IF NOT EXISTS (
    SELECT 1 FROM pg_constraint
    WHERE conname = 'opp_has_contact_or_account'
      AND conrelid = 'public.opportunities'::regclass
  ) THEN
    ALTER TABLE public.opportunities
      ADD CONSTRAINT opp_has_contact_or_account
      CHECK (contact_id IS NOT NULL OR account_id IS NOT NULL);
  END IF;
END $$;
```

**Section 9 — Idempotent data migration (verbatim from SEED-016, three steps):**
```sql
-- Step 1+2: distinct TRIM(company) per org → INSERT accounts (skip already-present)
WITH distinct_companies AS (
  SELECT org_id, TRIM(company) AS name
  FROM public.contacts
  WHERE company IS NOT NULL AND TRIM(company) <> ''
  GROUP BY org_id, TRIM(company)
)
INSERT INTO public.accounts (org_id, name, source, created_at, updated_at)
SELECT dc.org_id, dc.name, 'auto_from_contact_company', now(), now()
FROM distinct_companies dc
WHERE NOT EXISTS (
  SELECT 1 FROM public.accounts a
  WHERE a.org_id = dc.org_id AND lower(a.name) = lower(dc.name)
);

-- Step 3: link contacts.account_id where still NULL
UPDATE public.contacts c
SET account_id = a.id
FROM public.accounts a
WHERE a.org_id = c.org_id
  AND lower(TRIM(c.company)) = lower(a.name)
  AND c.account_id IS NULL;
```
