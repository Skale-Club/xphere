# Plan 42-02: OAuth Provider Pre-flight Configuration

## Objective

Configure Google OAuth credentials, enable the Google provider in Supabase Auth, and set required environment variables in Vercel. This plan is mostly **manual steps** — the executor guides the user through the external dashboards and verifies results via the Supabase client.

This plan runs in wave 1 in parallel with 42-01 (migration) because neither depends on the other.

**Important:** This plan has `autonomous: false` because it requires clicking through Google Cloud Console and Supabase Dashboard UI — Claude cannot do this autonomously.

## Tasks

### Task 1: Google Cloud Console — Create OAuth 2.0 Client

<read_first>
- CLAUDE.md (canonical production origin: `https://operator.skale.club`)
- .env.local (if exists — check for existing GOOGLE_ variables)
</read_first>

<action>
Guide the user through these steps in Google Cloud Console (https://console.cloud.google.com/):

1. **Create/Select Project:** Use existing GCP project or create one named "Operator"
2. **Enable OAuth consent screen:**
   - Go to APIs & Services → OAuth consent screen
   - User type: External (unless you have Google Workspace)
   - App name: "Operator"
   - User support email: skale.club@gmail.com
   - Developer contact: skale.club@gmail.com
   - Scopes: add `email` and `profile` (openid is automatic)
   - Test users: add skale.club@gmail.com for development
3. **Create OAuth 2.0 Client:**
   - Go to APIs & Services → Credentials → Create Credentials → OAuth client ID
   - Application type: Web application
   - Name: "Operator Web"
   - Authorized JavaScript origins:
     - `https://operator.skale.club`
     - `http://localhost:4267`
   - Authorized redirect URIs (these come from Supabase — you'll fill them in Task 2):
     - `https://<your-supabase-project-ref>.supabase.co/auth/v1/callback`
4. **Copy the credentials:**
   - Client ID (format: `XXXXXXXXXXX-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX.apps.googleusercontent.com`)
   - Client Secret (format: `GOCSPX-XXXXXXXXXXXXXXXXXXXXXXXXXX`)
</action>

<acceptance_criteria>
- User has Google OAuth Client ID and Client Secret values in hand
- Redirect URI is set to the Supabase project's callback URL
- OAuth consent screen is configured with `email` and `profile` scopes
</acceptance_criteria>

### Task 2: Supabase Dashboard — Enable Google OAuth Provider

<read_first>
- CLAUDE.md (NEXT_PUBLIC_SUPABASE_URL env var — derive the project ref from the URL, e.g. `https://xyzxyz.supabase.co` → project ref is `xyzxyz`)
</read_first>

<action>
In Supabase Dashboard (https://supabase.com/dashboard):

1. Open your project → Authentication → Providers
2. Find "Google" in the list → toggle Enable
3. Paste the Client ID and Client Secret from Task 1
4. Copy the "Callback URL (for OAuth)" shown in the Supabase UI — it looks like:
   `https://<project-ref>.supabase.co/auth/v1/callback`
5. Go back to Google Cloud Console and add this URL as an Authorized redirect URI for the OAuth client (if not already done in Task 1)
6. Save in Supabase
7. In Supabase Dashboard → Authentication → URL Configuration:
   - Site URL: `https://operator.skale.club`
   - Additional Redirect URLs: `http://localhost:4267`
</action>

<acceptance_criteria>
- Google provider shows "Enabled" in Supabase Auth Providers list
- Site URL in Supabase Auth is `https://operator.skale.club`
- `http://localhost:4267` appears in Additional Redirect URLs
</acceptance_criteria>

### Task 3: Vercel — Add NEXT_PUBLIC_SITE_URL environment variable

<read_first>
- CLAUDE.md (Vercel Hobby hosts the Next.js app; canonical production origin: `https://operator.skale.club`)
</read_first>

<action>
In Vercel Dashboard for the Operator project:

1. Go to Settings → Environment Variables
2. Add (if not present):
   - Name: `NEXT_PUBLIC_SITE_URL`
   - Value: `https://operator.skale.club`
   - Environments: Production, Preview, Development
3. Trigger a redeployment after adding env vars (or the next `git push` will pick them up)

Also add for local development in `.env.local` (do NOT commit this file):
```
NEXT_PUBLIC_SITE_URL=http://localhost:4267
```

Note: The Supabase Google provider credentials (Client ID and Secret) are stored IN Supabase — they do NOT need to be in Vercel env vars. The Supabase-managed OAuth flow handles them server-side.
</action>

<acceptance_criteria>
- `NEXT_PUBLIC_SITE_URL` is set in Vercel (Production + Preview + Development)
- `.env.local` contains `NEXT_PUBLIC_SITE_URL=http://localhost:4267`
- `.env.local` is in `.gitignore` (verify: `grep ".env.local" .gitignore`)
</acceptance_criteria>
