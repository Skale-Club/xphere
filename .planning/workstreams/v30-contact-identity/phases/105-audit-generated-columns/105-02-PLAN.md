**Probe A — generated columns populated on existing rows:**
```sql
SELECT
  count(*) AS total,
  count(phone_e164) AS with_phone_e164,
  count(email_normalized) AS with_email_normalized,
  count(*) FILTER (WHERE phone IS NOT NULL) AS with_raw_phone,
  count(*) FILTER (WHERE email IS NOT NULL) AS with_raw_email
FROM public.contacts;
```
Expected: `with_phone_e164` ≈ `with_raw_phone` (some raw phones may normalize to NULL e.g. all-non-digits); `with_email_normalized` ≈ `with_raw_email`.

**Probe B — identity_status backfill distribution:**
```sql
SELECT identity_status, count(*) FROM public.contacts GROUP BY identity_status ORDER BY count(*) DESC;
```
Expected: Only `identified` and `channel_only` values present. `channel_only` rows match D-03 predicate.

**Probe C — D-03 predicate equivalence:**
```sql
SELECT
  count(*) FILTER (WHERE identity_status = 'channel_only') AS marked_channel_only,
  count(*) FILTER (
    WHERE phone IS NULL AND email IS NULL
      AND source IN ('instagram','whatsapp','facebook','messenger')
      AND external_id IS NOT NULL
  ) AS predicate_match
FROM public.contacts;
```
Expected: `marked_channel_only = predicate_match` exactly.

**Probe D — normalize_phone() equivalence table (10 cases):**
```sql
SELECT input, public.normalize_phone(input) AS got, expected FROM (VALUES
  (NULL::text,                       NULL::text),
  ('',                               NULL),
  ('   ',                            NULL),
  ('+1 (555) 010-0100',              '+15550100100'),
  ('555-0100',                       '5550100'),
  ('+',                              NULL),
  ('++1234',                         '+1234'),
  ('abc',                            NULL),
  ('  +44 20 7946 0958  ',           '+442079460958'),
  ('+1-555-0100 ext. 42',            '+15550100 42'::text)  -- record actual; oversized normalization is intentional (Pitfall 7)
) AS t(input, expected);
```
Expected: `got = expected` for rows 1-9 (row 10 is a deliberate Pitfall 7 documentation case — record but do not assert).

**Probe E — CHECK constraint rejects invalid value:**
```sql
-- Run inside a savepoint to avoid leaking test row
DO $$ BEGIN
  BEGIN
    INSERT INTO public.contacts (org_id, identity_status)
    VALUES ('00000000-0000-0000-0000-000000000000', 'nonsense');
    RAISE EXCEPTION 'CHECK constraint did NOT reject invalid value';
  EXCEPTION WHEN check_violation THEN
    RAISE NOTICE 'CHECK constraint correctly rejected nonsense (SQLSTATE 23514)';
  END;
END $$;
```
Expected: NOTICE message; no rows inserted.

**Probe F — refresh_contact_duplicate_audit() runs:**
```sql
SELECT public.refresh_contact_duplicate_audit();
SELECT match_type, count(*) AS clusters, sum(cluster_size) AS dupe_rows, max(cluster_size) AS largest
FROM public.contact_duplicate_audit
GROUP BY match_type;
```
Expected: Returns rows (may be zero clusters if branch has no duplicates — acceptable).

**Probe G — oversized normalized values (Pitfall 7 surface):**
```sql
SELECT match_type, normalized_value, cluster_size, length(normalized_value) AS len
FROM public.contact_duplicate_audit
WHERE length(normalized_value) > 20
ORDER BY len DESC
LIMIT 20;
```
Expected: May return rows (Phase 106 fodder). Record count.

**Probe H — RLS on contact_duplicate_audit unchanged from pattern:**
```sql
SELECT polname, cmd, qual::text FROM pg_policies
WHERE schemaname='public' AND tablename='contact_duplicate_audit';
```
Expected: One row, polname `contact_duplicate_audit_select`, cmd `SELECT`, qual contains `get_current_org_id`.

**Probe I — RLS on contacts unchanged (success criterion #8):**
```sql

WHERE schemaname='public' AND tablename='contacts'
ORDER BY polname;
```
