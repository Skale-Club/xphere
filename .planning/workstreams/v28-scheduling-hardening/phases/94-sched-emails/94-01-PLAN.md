# Plan 94-01 — Resend client + sendBookingConfirmation

## Objective

Create `src/lib/scheduling/emails.ts` exporting `sendBookingConfirmation` that emails the booker a confirmation with event details + cancel link.

## Tasks

### Task 1 — emails.ts module (auto)

Create `src/lib/scheduling/emails.ts`:

- Singleton Resend client wrapped in a getter that returns `null` when `RESEND_API_KEY` is missing
- `sendBookingConfirmation(params)` accepting:
  - bookerEmail, bookerName
  - hostName
  - eventTitle
  - startAt (Date or ISO), endAt (Date or ISO)
  - timezone (IANA)
  - cancelUrl (string)
- Returns `Promise<void>`. NEVER throws — try/catches everything and logs a warning on failure.
- HTML template inline: dark theme (`#08090A` bg, `#FAFAFA` text), table layout, event details, cancel button styled like an indigo button.
- Subject: `Booking confirmed: {eventTitle} with {hostName}`
- From: `Xphere Scheduling <bookings@xphere.skale.club>` (override via `process.env.RESEND_FROM` if set)
- Date formatting uses date-fns + date-fns-tz to render the start time in the booker's locale string in their timezone.

## Verification

- `src/lib/scheduling/emails.ts` exists and exports `sendBookingConfirmation`
- Function returns `void` Promise even when `RESEND_API_KEY` is unset (no-op + warning)
