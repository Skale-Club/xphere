# Plan 95-01 — Required custom fields defaults in scheduling contact create

## Objective

Pre-fill defaults for required `custom_field_definitions` when `createBooking` auto-creates a contact. If validation/insert still fails, fall back to no-contact-link without throwing.

## Tasks

### Task 1 — Defaults helper + integration (auto)

In `src/app/(dashboard)/scheduling/_actions/bookings.ts`:

1. Add a helper `buildRequiredCustomFieldDefaults(orgId)` that queries `custom_field_definitions` where `entity='contact'`, `required=true`, `archived=false`. Returns a `Record<string, unknown>` keyed by `key` with a type-appropriate default:
   - `text`, `long_text`, `select`, `email`, `url`, `phone`: `''`
   - `number`, `integer`: `0`
   - `boolean`: `false`
   - `date`: `'1970-01-01'`
   - `datetime`: `'1970-01-01T00:00:00.000Z'`
   - `multi_select`: `[]`
   - `currency`: `{ amount: 0, currency: 'USD' }`
2. Use this helper in the existing contact-create block. Merge with `{}` payload (no per-booker custom values are collected yet — purely defaults to satisfy required-field constraint).
3. Wrap the insert (`contacts.insert(...)`) in try/catch; on validation failure, log warning + leave `linkedContactId = null`. Booking still proceeds.

## Verification

- `buildRequiredCustomFieldDefaults` defined and called in `createBooking`
- contact insert payload includes `custom_fields: <defaults>`
- failure path logs warning and continues to booking insert
- `npm run build` exits 0
