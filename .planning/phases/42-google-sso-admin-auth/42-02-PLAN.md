  <instructions>
Per D-42-13 Pre-Execution Checklist:

**Step A — Google Cloud Console (create OAuth 2.0 Web client):**
   1. Visit https://console.cloud.google.com → select (or create) the Operator project
   2. APIs & Services → Credentials → "+ Create Credentials" → "OAuth client ID"
   3. Application type: **Web application**
   4. Name: "Operator (Supabase Auth)"
   5. Authorized JavaScript origins: `https://operator.skale.club` (optional add `http://localhost:4267` for local)
   6. Authorized redirect URIs: paste the EXACT Supabase callback URL:
      `https://<your-supabase-project-ref>.supabase.co/auth/v1/callback`
      (replace `<your-supabase-project-ref>` with the real ref visible in Supabase Dashboard → Settings → API)
   7. Create → copy the **Client ID** and **Client Secret** (you will not see the secret again)

**Step B — Supabase Dashboard (enable Google provider):**
   1. Visit Supabase Dashboard → Authentication → Providers → Google
   2. Toggle "Enable Sign in with Google" ON
   3. Paste **Client ID** and **Client Secret** from Step A
   4. Authentication → URL Configuration:
      - Site URL: `https://operator.skale.club`
      - Redirect URLs allowlist: add `https://operator.skale.club/auth/callback` AND (optional) `http://localhost:4267/auth/callback`
   5. Save

**Step C — Vercel (env var):**
   1. Visit Vercel Dashboard → Operator project → Settings → Environment Variables
   2. Add `NEXT_PUBLIC_GOOGLE_OAUTH_CLIENT_ID` = (Client ID from Step A)
   3. Apply to: Production AND Preview (do NOT add to Development unless you want local OAuth)
   4. Save (no redeploy needed yet — next push picks it up)

**Step D — Smoke test (optional but recommended):**
   1. In Supabase Dashboard → Authentication → Users → "Add user" → "Send invitation" is NOT what we want; instead verify the provider UI shows Google as enabled (green dot)
   2. Visit https://supabase.com/docs/guides/auth/social-login/auth-google#configure-the-google-provider — confirm steps match what you did
  </instructions>
