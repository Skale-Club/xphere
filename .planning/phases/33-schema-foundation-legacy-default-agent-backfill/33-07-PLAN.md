<objective>
Single-checkpoint plan that finalizes Phase 33: operator runs `npx supabase db push` to apply the 7 schema migrations (034-040) to the linked Supabase project, then the executor regenerates `src/types/database.ts` and verifies `npm run build` passes. Without it, Phase 34 (Runtime Skeleton) would start building against types that don't exist yet, and tests in `tests/agent-schema-*.test.ts` would remain RED forever.

Instead, this plan asserts the schema + seed preconditions that Phase 35's GATE-07 backfill needs to succeed: every org has a Main Agent, every Main Agent has a web_widget channel default, every active tool_config is granted to its org's Main Agent, and the prompt byte-equality holds.

Output: 1 regenerated TS file + 1 successful build + a captured spot-check report in the plan summary.
</objective>
