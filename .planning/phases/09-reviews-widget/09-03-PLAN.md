  <action>
    Create `src/components/reviews/review-widget-configurator.tsx` as a client component.

    Required UX:
    1. Accept props for one location:
       - `locationId`
       - `locationName`
       - `reviewToken`
       - `mapsUrl`
       - `reviews` (already cached rows from Supabase)
    2. Render controls for:
       - layout selector (carousel, grid, list, compact)
       - theme selector (light, dark)
       - primary color
       - star color
       - show/hide photo
       - show/hide date
       - show/hide Google button
       - border radius
       - max width
    3. Build the final embed snippet in-memory using `https://operator.skale.club/reviews-widget.js` and the location's `review_token`.
    4. Provide a copy button using `navigator.clipboard.writeText(...)` with `sonner` success/error toasts.
    5. Render a lightweight local preview using the cached review rows so admins can inspect the chosen layout before copying.

    Important design decision:
    - Persist nothing to the database in Phase 9.
    - The chosen appearance lives in the copied script tag `data-*` attributes, which keeps the phase aligned with the already-shipped schema from Phase 7.
  </action>
