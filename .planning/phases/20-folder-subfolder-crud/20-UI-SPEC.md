## Interaction Contracts

### IC-01: Folder Collapse Toggle

- **Trigger:** Click the `ChevronRight`/`ChevronDown` icon at the left of the folder header (leftmost interactive element, before the GripVertical handle).
- **Default state:** All folders expanded on mount.
- **Collapsed state:** Tool rows and subfolder rows under this folder are hidden (CSS `hidden` class toggled via `collapsedFolders` Set state). The chevron rotates from `ChevronRight` to `ChevronDown`.
- **Icon sizes:** `h-3.5 w-3.5` ŌĆö matches existing `GripVertical` size.
- **Subfolder collapse:** Each subfolder also has its own collapse toggle. Collapsing a parent folder hides all child content including subfolders.

### IC-02: Inline Rename (Folders and Subfolders)

- **Trigger:** Click the folder/subfolder name label text. The label `<span>` is replaced with an `<Input>` via conditional render based on `renamingFolderId === folder.id`.
- **Input behavior:**
  - `autoFocus` ŌĆö cursor placed at end of existing name on mount.
  - `Enter` key ŌåÆ `commitRename(folderId)` ŌĆö trims value, calls `updateFolder`, dismisses input.
  - `Escape` key ŌåÆ cancel, restore original label, no server call.
  - `onBlur` ŌåÆ same as `Enter` (commit), except when blur target has `data-rename-cancel="true"` attribute (prevents double-fire when clicking an action button).
  - Empty string on commit ŌåÆ treated as cancel (no server call, original name restored).
- **State:** `renamingFolderId: string | null` and `renameValue: string` in `ToolsTable`. Single pair handles both folder and subfolder levels (IDs are globally unique UUIDs).
- **Loading feedback:** No spinner on inline rename. `toast.success('Folder renamed.')` on success; `toast.error(message)` on failure.

### IC-03: Add Subfolder via (+) Button

- **Trigger:** Click the `Plus` icon button on a folder header (visible on hover via `opacity-0 group-hover:opacity-100`).
- **Hover visibility class:** `opacity-0 group-hover:opacity-100 transition-opacity` on the button. The TableRow must have `className="group"`.
- **Form placement:** An extra `TableRow` appears immediately below the triggering folder header (before its subfolders and tools), with `colSpan={columns.length}` and `pl-10` left padding.
- **Form elements:** `Input` (`h-7 w-36 text-xs`, placeholder "Subfolder name") + "Add Subfolder" `Button` (`size="sm" variant="outline" h-7 text-xs`) + "Cancel" `Button` (`size="sm" variant="ghost" h-7 text-xs`).
- **Escape in input** ŌåÆ cancel, dismiss form row.
- **Submit (Enter or "Add Subfolder" button):** Calls `createFolder(name, parentFolderId)`, then `router.refresh()`. `toast.success('Subfolder created.')` on success.
- **State:** `addingSubfolderTo: string | null` (parent folder ID) and `newSubfolderName: string` in `ToolsTable`.
- **Constraint:** Only one add-subfolder form visible at a time. Opening a second one closes the first.

### IC-04: Add Top-Level Folder (Wire Existing Stub)

- **Trigger:** Click the `FolderPlus` toolbar button (existing). Existing form UI is already rendered.
- **Change from Phase 19:** Wire the `handleAddFolder` form submit to call `createFolder(name, null)` server action, then `router.refresh()`. No UI changes to the form.
- **Feedback:** `toast.success('Folder created.')` on success; `toast.error(message)` on failure.

### IC-05: Delete Folder / Subfolder (Two-Option Modal)

- **Trigger:** Click the `Trash2` icon button on a folder or subfolder header (hover-reveal, same pattern as rename).
- **Modal:** `AlertDialog` with `open={!!folderDeleteTarget}`.
- **Title:** `Delete "{folderName}"?` (folder name interpolated).
- **Description:** `Choose what happens to the tools inside this folder.`
- **Three buttons in footer (`AlertDialogFooter className="flex-col sm:flex-row gap-2"`):**
  1. `AlertDialogCancel` ŌĆö "Cancel" (no action)
  2. `AlertDialogAction className={cn(buttonVariants({ variant: 'outline' }))}` ŌĆö "Move tools to Ungrouped" ŌåÆ calls `deleteFolder(id)` (DB ON DELETE SET NULL handles orphaning)
  3. `AlertDialogAction className="bg-destructive text-destructive-foreground hover:bg-destructive/90"` ŌĆö "Delete folder and tools" ŌåÆ calls `deleteFolderWithTools(id)`
- **State:** `folderDeleteTarget: { folder: ToolFolder } | null` in `ToolsTable`.
- **After action:** `toast.success('Folder deleted.')` or `toast.error(message)`. State cleared on close.

### IC-06: Folder Select in Tool Config Form

- **Location:** `tool-config-form.tsx`, new `<FormField>` for `folder_id`.
- **Label:** "Folder" with `(optional)` in `text-muted-foreground font-normal`.
- **Default / empty value:** Sentinel string `'__none__'` maps to `null` in DB. `<SelectItem value="__none__">No folder</SelectItem>` as first item.
- **Visual hierarchy in list:** Top-level folders render as plain `SelectItem`. Subfolders render as indented `SelectItem` with `&nbsp;&nbsp;` prefix before the name (no deeper nesting ŌĆö max 2 levels).
- **Submit transform:** `folder_id: values.folder_id === '__none__' ? null : values.folder_id`.
- **Zod schema update:** `folder_id: z.string().optional()` (not `.uuid()` ŌĆö sentinel `'__none__'` is not a UUID). UUID validation deferred to server action.

## Subfolder Visual Treatment

**Indentation approach:** `pl-8` (32px) on the `<TableCell colSpan={columns.length}>` of `SubfolderHeader` rows. Top-level folder headers use `px-4` (16px). This 16px additional indent clearly communicates parent-child relationship without a full tree widget.

**SubfolderHeader row structure:**
```
TableRow [className="bg-muted/30 hover:bg-muted/40 group"]
  TableCell [colSpan=N, className="py-1.5 pl-8 pr-4"]
    div [className="flex items-center gap-2"]
      ChevronRight/ChevronDown [h-3.5 w-3.5, collapse toggle]
      span OR Input [label or rename input]
      span [tool count "(N)", text-xs text-muted-foreground]
      div [className="ml-auto flex items-center gap-1 opacity-0 group-hover:opacity-100 transition-opacity"]
        Button [variant="ghost", size="icon", Pencil icon ŌĆö rename]
        Button [variant="ghost", size="icon", Trash2 icon ŌĆö delete]
```

**No GripVertical handle on SubfolderHeader:** Phase 21 owns subfolder DnD. The grip spacer (`<span className="w-3.5" />`) from `StaticFolderHeader` is used instead to maintain alignment.
