# Phase 107: Event Dispatch — Plan

## Goal
Wire calendar booking events to trigger workflow execution through the unified engine. Pipeline events are already fully wired — this closes the calendar gap.

## Background
- `src/lib/pipeline/events.ts` already works: `emitOpportunityEvent()` → `findMatchingWorkflows()` → `buildOpportunityScope()` → `recordDispatch()` → **`runFlowSync()`** (fire-and-forget) ✅
- `src/lib/scheduling/transition.ts` has `emitCalendarEvent()` but it **never invokes the workflow runtime** — it records the dispatch row and stops ❌
- `buildMeetingScope()` in `scope.ts` exists but is never called by the event dispatch path
- No booking server action (`createBooking`, `cancelBooking`, `cancelBookingByToken`) calls any `transition.ts` function
- The transition.ts functions (`confirmBooking`, `cancelBooking`, `markNoShow`, `rescheduleBooking`) are orphan code — never imported anywhere
- `MAX_CASCADE_DEPTH=3` guard already exists in `transition.ts`

## Changes

### 1. Wire `emitCalendarEvent()` to dispatch workflows (`src/lib/scheduling/transition.ts`)
Mirror the `emitOpportunityEvent()` pattern:
- Update `findMatchingWorkflows()` to also return `current_version_id`
- Import `runFlowSync`, `createServiceRoleClient`, `buildMeetingScope`
- After recordDispatch(), build meeting scope and fire `runFlowSync()` for each matched workflow
- Fire-and-forget with `.catch()` on each dispatch
- The calling paths (cron route, booking actions) already pass a service-role supabase client

### 2. Wire `createBooking()` → `meeting.scheduled` (`src/app/(dashboard)/scheduling/_actions/bookings.ts`)
- Import `emitCalendarEvent` from `@/lib/scheduling/transition`
- After successful booking insert (line 483), create a service-role `TransitionContext` and emit `meeting.scheduled`

### 3. Wire `cancelBooking()` → `meeting.cancelled` (`bookings.ts`)
- After successful cancellation, emit `meeting.cancelled`

### 4. Wire `cancelBookingByToken()` → `meeting.cancelled` (`bookings.ts`)
- After successful cancellation, emit `meeting.cancelled`
