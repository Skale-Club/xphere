For each of the 26 listed files, perform the following edits. SKIP `src/app/(dashboard)/error.tsx` entirely (it has no breadcrumb context).

**Step 1 — Read each file individually** before editing. Pattern-matching across files is brittle because each page wraps the title differently (flex containers, grids, tabs, etc.).

**Step 2 — Remove these blocks when found:**

(a) The static title + description pair, typical shape:
```tsx
<div>
  <h1 className="text-[28px] sm:text-[32px] font-semibold tracking-tight text-text-primary">
    Page Name
  </h1>
  <p className="mt-1 text-[14px] text-text-secondary">Static description.</p>
</div>
```
Class names may vary slightly (e.g., different responsive sizes, additional wrapper classes). The signal is: `<h1>` with the `text-[28px] sm:text-[32px]` (or close variants like `text-2xl`/`text-3xl` paired with `font-semibold tracking-tight`) hard-coded as a section header, optionally followed by a sibling `<p>` description.

(b) Eyebrow blocks IMMEDIATELY above the `<h1>`:
```tsx
<div className="flex items-center gap-2 text-[12px] uppercase tracking-[0.08em] text-text-tertiary">
  <Icon className="h-3.5 w-3.5" />
  <span>SECTION NAME</span>
</div>
```
Look for `uppercase tracking-[0.08em]` + small icon as the signal.

**Step 3 — Preserve container & siblings:**
- If the title block sits inside a flex container alongside action buttons / tabs / filters, KEEP the buttons and KEEP the container, but:
  - If the container is now empty after removing the title block → remove the container too.
  - If the container used `justify-between` and now only has the right-side group → change to `justify-end` (or remove `justify-between` if natural).
  - If the container had `mb-*` or `gap-*` that no longer makes sense, leave layout tokens alone unless the rendered output would clearly look broken — defer visual judgment, don't over-edit.

**Step 4 — Dynamic detail pages — preserve dynamic h1:**
For the following files, the `<h1>` is dynamic (renders an entity name, opportunity name, order number, conversation subject, etc.):

- Remove only the **static** description `<p>` adjacent to it (and any eyebrow above it).
- If the h1 is actually static text (no interpolation), remove it like the others.
- Do NOT introduce `useBreadcrumbOverride()` calls in this pass — out of scope.

**Step 5 — Clean unused imports:**
After removing eyebrow blocks, check imports. If a `lucide-react` icon was imported ONLY for the eyebrow (i.e., not used elsewhere in the file after the edit), remove it from the import statement. If the import statement becomes empty, remove it entirely. Same applies to any other import that becomes unused (e.g., a `Calendar` icon used only in the eyebrow).

**Step 6 — Do NOT touch:**

- Layout grids, page containers, spacing tokens unrelated to the removed block.
- Styling tokens or design-system primitives.
- Server-component data fetching, hooks, or state.
