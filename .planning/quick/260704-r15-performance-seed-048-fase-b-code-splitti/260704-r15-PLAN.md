**Critical Next.js 16 constraint (verified against current docs):**
`ssr: false` passed to `next/dynamic` is **only allowed inside a Client Component
file** ('use client' at the top). Calling `dynamic(() => import(...), { ssr: false })`
directly inside a Server Component throws a build error: "ssr: false is not allowed
with next/dynamic in Server Components. Please move it into a Client Component."
`src/app/(dashboard)/calls/(hub)/page.tsx` is a Server Component (async function,
no 'use client'), and `src/components/calls/call-detail-human.tsx` is also an
async Server Component. Both need a small 'use client' wrapper file that does the
`dynamic(..., { ssr: false })` call — the Server Component then imports that
wrapper like any other component. This is the exact pattern already used in this
codebase: `src/components/design-system/confetti-burst.tsx` has 'use client' at
the top and does `const Confetti = dynamic(() => import('react-confetti'), { ssr: false })`.
