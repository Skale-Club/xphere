### What landed

- **Migration 1044** (`supabase/migrations/1044_project_folders.sql`): `project_folders` table with self-FK on `parent_id`, RLS via `get_current_org_id()`, sibling index, and `updated_at` trigger. Adds `folder_id`, `position`, `archived_at`, `deleted_at` to `projects` with partial indexes for folder / archived / deleted.
- **Types** (`src/types/database.ts`): Extended `ProjectRow` with the 4 lifecycle columns; added `ProjectFolderRow`.
- **Folder server actions** (`src/app/(dashboard)/projects/_actions/folders.ts`): `listFolders`, `createFolder`, `renameFolder`, `updateFolderMeta`, `reorderFolders`, `moveFolder`, `archiveFolder`, `deleteFolder` — direct port of the workflows version with `workflow_folders → project_folders` and the cascade hitting the `projects` table.
- **Project lifecycle actions** (extended `src/app/(dashboard)/projects/actions.ts`): `moveProjectToFolder`, `reorderProjectsInFolder`, `archiveProject`, `unarchiveProject`, `softDeleteProject`, `restoreProjectFromTrash`, `hardDeleteProject`, `emptyProjectsTrash`. `getProjects()` now accepts `{ includeArchived, includeDeleted }` and orders by `position ASC, created_at DESC`.
- **UI components**:
  - `NewFolderButton` (`src/components/projects/new-folder-button.tsx`) — port of the workflows version, copy adjusted to "projects".
  - `ProjectsListFolders` (`src/components/projects/projects-list-folders.tsx`) — full DnD list with collapsible Unfiled + folder groups, sortable rows, three-column row layout (NAME with color dot / STATUS Active|Archived / UPDATED relative time), row `[...]` menu (Move to folder · Archive/Unarchive · Move to trash), folder header `[...]` menu (Rename · Delete). TRIGGER_META, WorkflowToggle and is_active guards intentionally removed. `TouchSensor` is wired alongside `PointerSensor` (P11 mobile polish).
- **/projects page rewrite** (`src/app/(dashboard)/projects/page.tsx`): toolbar mirrors `/workflows` minus the Logs button. LEFT cluster = `+ Project` then `Folder` (R01 placement preserved — verified via the in-plan `node -e` check). RIGHT cluster = Show archived toggle + Trash link with badge. Mobile dropdown carries the same right-cluster items. Reads projects via `getProjects({ includeArchived })`, folders + trash count in parallel via Supabase client.
- **/projects/trash route** (`src/app/(dashboard)/projects/trash/page.tsx` + `trash-actions.tsx`): mirrors workflows trash. Lists soft-deleted projects with color dot + name + relative deleted time. Per-row Restore / Delete forever; page-level Empty trash. Wired to `restoreProjectFromTrash`, `hardDeleteProject`, `emptyProjectsTrash`.
