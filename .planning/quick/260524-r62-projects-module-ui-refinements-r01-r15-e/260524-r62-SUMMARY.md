## Deviations / Notes

1. **R12 db push:** see Migration section. The `npx supabase db push` step was not completed by this agent due to a **pre-existing** migration-history mismatch unrelated to R-62. Treated per plan rule: commit the migration file and surface for manual push. No code regressions because consumers handle a `null` assignee gracefully.
2. **R03 default step:** the dialog originally had `defaultStep = 'backlog'` as a default value, which made the "Will be added to" hint always render. R-62 changes this to render the hint **only when the caller explicitly passes a `defaultStep`** (matches the Board column "Add task" affordance) while the page-header "Add Task" button (no `defaultStep`) gets no hint and creates in `backlog`.
3. **R10 + R12 footer ordering:** with both the due-date span and the assignee avatar in the TaskCard footer, the avatar is anchored at the right edge. When no `end_date` is present, the avatar takes the `ml-auto` to push itself to the right; when both are present, the date keeps `ml-auto` and the avatar follows it.
4. **R02 apostrophe:** "You don't have any projects yet" is rendered as `You don&apos;t have any projects yet` because `react/no-unescaped-entities` would otherwise fail the build. Text content is semantically identical.
